# Public Release Notes

这个仓库已经调整为更适合公开分享的形态：

- 不跟踪本地数据库 `data/mfa-local.db`
- 不跟踪打包产物和测试 exe
- 程序启动时不再依赖仓库内置数据库
- 首次无数据时，优先让用户导入你发布的数据快照
- `同步官网` 保留为备用入口，而不是默认路径

## 推荐发布结构

1. GitHub 仓库：源码
2. GitHub Release 附件一：不内置数据库的 exe
3. GitHub Release 附件二：数据快照

## 快照命名建议

- `mfa-local-snapshot-2026-07-05.db`
- `mfa-local-snapshot-2026-07-05.zip`

建议把提取日期直接写进文件名，并在 Release 里补充：

- 快照提取日期
- 条目总数
- 数据来源说明
- 非官方产品说明

## 发布前检查

1. 仓库里没有 `exe`、`.db`、`bin/`、`obj/` 等产物被纳入提交。
2. 你是否要附带 `LICENSE`。
3. 你是否已经本地验证：空库启动、导入快照、正常检索、备用同步路径可用。
